# Code Review Skill

## 触发条件
当用户要求审查代码、review PR 或检查代码质量时激活。

## 操作流程

### 第一步：收集上下文
1. 使用 read_file 读取目标文件完整内容
2. 识别文件所属模块和依赖关系

### 第二步：多维度审查
按以下维度逐项检查：

| 维度 | 检查项 | 严重程度 |
|------|--------|----------|
| 正确性 | 逻辑错误、空指针、类型不匹配 | CRITICAL |
| 安全性 | 注入、硬编码密钥、权限绕过 | CRITICAL |
| 性能 | N+1查询、不必要的拷贝、资源泄漏 | WARNING |
| 可维护性 | 过长函数、重复代码、缺失文档 | INFO |
| 风格 | 命名规范、格式一致性 | INFO |

### 第三步：输出审查报告
格式要求：
```
## 审查结果：{文件路径}

### [CRITICAL] {问题标题}
- **位置**：第 X 行
- **问题**：{描述}
- **建议**：{修复方案}
```

## 约束
- 不要修改代码，只输出审查结果
- 每个问题必须附带文件路径和行号
- 安全问题统一标记为 [CRITICAL]
- 如果没有发现问题，明确说明"未发现问题"
